# Support

## Routing

- Repo-shape or documentation issues: open an issue or PR in this repo.
- Evidence disputes: use the `evidence_dispute` issue template.
- Security problems: follow [../SECURITY.md](../SECURITY.md).
- Live engineering questions: route through the active session owner instead of
  editing their work in place.

## Review Expectation

Support requests should point to exact files, receipts, or commands. Narrative
without a concrete artifact is not enough for DM3.
