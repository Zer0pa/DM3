# Reset Basis

## Why A Reset Is Needed

The surviving evidence shows at least two different DM3 strata:

- an older, source-available dual-meru / DEQ / resonance / proof stack
- a later, source-missing hybrid stack that added GPU kernels, micro-transformer layers, HRM bridging, tagging, and phoneme assets

That means the current problem is not just "missing files." It is also a scientific and architectural drift problem.

## Carry Forward Without Re-arguing

These elements are strong enough to carry forward as real prior work:

- dual-meru geometry as an actual implemented substrate
- DEQ / settling style updates
- resonance and proof-gate thinking
- deterministic artifact and receipt discipline
- the idea that transformer-like modules were meant to be embedded in the geometry, not define the whole system

## Re-open From First Principles

These points should be reopened instead of inherited uncritically:

- what the authority metric actually is
- what physical claim the manifold is supposed to support
- whether the geometry is a substrate, an inductive bias, or the computation itself
- how incoming data is encoded onto the manifold
- whether the HRM / transformer additions are readout helpers or central dynamical components
- what counts as success beyond "interesting behavior"

## Hard Separation For The Restart

The restart should separate the stack into four layers:

1. Geometry
   - dual-meru mesh
   - adjacency
   - region / ring / boundary tags

2. Dynamics
   - settling / DEQ
   - resonance
   - control terms
   - energy or stability functions

3. Learned Modules
   - micro-transformer or attention blocks
   - HRM bridge, if retained
   - readout, memory, classifier layers

4. Acceptance
   - deterministic geometry checks
   - limiting-case and stability checks
   - dataset / asset schema checks
   - authority metric for the actual scientific claim

## Immediate Scientific Rebase Questions

Before rebuilding the missing hybrid layer, answer these explicitly:

- What is the minimal scientific claim of DM3?
- What is the decisive observable or acceptance gate?
- What is the manifold supposed to compute that a baseline transformer does not?
- What is the data-to-manifold encoding rule?
- What settles, and under what conserved or monitored quantities?
- Which experiments are diagnostic, and which are only interesting side behavior?

## Engineering Implication

The restart should begin from the recoverable geometry / DEQ / resonance substrate and only reintroduce hybrid learned layers behind explicit interfaces, tests, and machine-readable experiment logging.

No restart step should depend on unrecovered intuition alone.
