# Heterogeneous Embodiment Prep Runbook

## Exact Phase Verdict

- `phase_outcome=PASS`
- `route_outcome=FAIL`
- The preserved bundled G2 family is retired on the current callable same-binary surface.
- The blocker is no longer "which same-binary route should we try?"; it is whether the historical bundled G2 residue belongs to a neighboring launcher generation or requires explicit redevelopment.

## Purpose

Prepare a bounded heterogeneous-compute test surface without changing the
scientific object or authority metric.

## Current Lane Status

- current status: premature
- heterogeneous embodiment remains an implementation hypothesis, not a branch
  claim
- the retired same-binary bundled `G2` family cannot be used as the reference
  observable

## Preconditions

Heterogeneous embodiment testing is allowed only after:

1. a single-lane CPU reference observable is already live and receipted
2. GPU parity has already passed on that same observable
3. NPU feasibility is either negative or explicitly bounded to a narrow assist
   role
4. the split preserves the same authority metric and claim ceiling

If any one of these is missing, heterogeneous work is premature.

## Required Handoff Doctrine

Every heterogeneous attempt must name:

1. which lane owns scheduling and final verdicts
2. which lane owns bounded relaxation work
3. which lane, if any, owns bounded projection work
4. the artifact path for each handoff
5. the final observable compared against the CPU-only reference

## Serious-Run Inheritance

Any live heterogeneous run must inherit:

- the identity packet from `RUNBOOK_RM10_SYSTEMS.md`
- the manifest and outcome encoding from `COMET_SCHEMA_AND_TAGGING.md`
- the checkpoint contract from `CHECKPOINT_RESUME_POLICY.md`

## Kill Criteria

Stop immediately if:

- the split erases the comparable observable
- one lane becomes a black box
- drift cannot be localized to one side of the split
- the split changes the scientific story instead of the execution plan
- mixed-lane language starts standing in for architecture proof

## Output Package

The heterogeneous package must end with:

- CPU reference receipt bundle
- split-lane receipt bundle
- explicit handoff artifacts
- runtime identity for every participating lane
- drift-localization note
- explicit `authority_status`, `evidence_surface`, `build_class`,
  `phase_outcome`, and `route_outcome`
- final verdict: `PASS`, `FAIL`, or `ABSTAIN`

## Current Status Sentence

Heterogeneous embodiment is premature; because `phase_outcome=PASS` and
`route_outcome=FAIL` retired the preserved bundled G2 family on the current
callable same-binary surface, no mixed-lane test may start until a single-lane
CPU reference observable is already live, GPU parity has passed on that same
observable, and NPU feasibility is already bounded.
