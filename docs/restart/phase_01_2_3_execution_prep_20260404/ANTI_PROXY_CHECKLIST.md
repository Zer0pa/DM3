# Anti-Proxy Checklist

## Exact Phase Verdict

- `phase_outcome=PASS`
- `route_outcome=FAIL`
- The preserved bundled G2 family is retired on the current callable same-binary surface.
- The blocker is no longer "which same-binary route should we try?"; it is whether the historical bundled G2 residue belongs to a neighboring launcher generation or requires explicit redevelopment.

Use this checklist before approving any plan, run, memo, or verdict that
touches the post-close handoff.

## Same-Binary Retirement Discipline

- [ ] Are we refusing to narrate another same-binary route sweep as new
      progress?
- [ ] If the historical bundled `G2` residue is mentioned, do we say whether
      the question is neighboring-launcher provenance or explicit
      redevelopment?
- [ ] Are we preserving `phase_outcome=PASS` and `route_outcome=FAIL` instead
      of blending them into a softer story?

## Smoke And Strings Proxies

- [ ] Are we explicitly distinguishing launch-proof smoke from substantive task
      recovery?
- [ ] If a run normalized to the smoke canonical `d3e721...`, did we classify
      it as smoke instead of progress?
- [ ] Are task names, strings, preserved outputs, or help text being treated
      only as clues, not as proof of a live route?
- [ ] Are wrapper names or asset filenames prevented from being narrated as
      recovered semantics?

## Governance And Lane Proxies

- [ ] Are we separating explicit-hash validation from repaired default canonical
      validation?
- [ ] Are we preventing fresh RM10 Genesis executability from being described
      as sovereign authority?
- [ ] Are we keeping fresh RM10 Genesis below source-built parity until that is
      actually proved?
- [ ] Does every comparison name whether the lane is `sovereign`,
      `governed_non_sovereign`, `engineering_only`, or `feasibility_only`?

## Hidden Redevelopment

- [ ] Is every proposed archaeology step bounded to provenance, launcher,
      wrapper, cwd, env, asset pairing, or inspection?
- [ ] If missing logic would need to be recreated, are we classifying that as
      redevelopment instead of archaeology?
- [ ] Are we forbidding modern substitute code from being passed off as
      recovery?

## Hardware Inflation

- [ ] Are we preventing GPU presence from being narrated as GPU readiness?
- [ ] Are we preventing NPU-adjacent visibility from being narrated as a live
      assist path?
- [ ] Are we preventing heterogeneous possibility from being narrated as an
      embodiment result before prerequisites are met?

## Supplement Vs Canon

- [ ] If the SoC training supplement is cited, is it labeled
      historical/contextual only?
- [ ] Are we refusing to import its thresholds or CPU/GPU/NPU role claims as
      current canon?
- [ ] Does current canon still win wherever the supplement says more than the
      present evidence supports?

If any item above is `No`, the work is not canon-clean.
