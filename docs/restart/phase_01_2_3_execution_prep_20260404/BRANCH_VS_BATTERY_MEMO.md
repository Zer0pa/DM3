# Branch Vs Battery Memo

## Exact Phase Verdict

- `phase_outcome=PASS`
- `route_outcome=FAIL`
- The preserved bundled G2 family is retired on the current callable same-binary surface.
- The blocker is no longer "which same-binary route should we try?"; it is whether the historical bundled G2 residue belongs to a neighboring launcher generation or requires explicit redevelopment.

## Purpose

This memo stops later execution from confusing scientific branches with battery
classes, device lanes, launcher variations, or hardware ambition.

## Governing Rule

Create or discuss a real branch only when the alternative changes the
scientific or architectural interpretation of DM3.
Do not create a new branch merely because:

- the run moved from Mac CPU to RM10 CPU
- the run moved from CPU to GPU
- the run added or removed an NPU assist attempt
- the run became heterogeneous
- the battery got longer
- a validator rule changed
- a launcher, wrapper, cwd, env, or asset variation was tried

## Current Branch Surface

The live scientific branches remain the already-frozen object-level questions:

- field computation versus weaker substrate readings
- boundary-transformer and nodewise continuation versus stronger center-model
  claims
- device-lane significance only if the phone becomes strong enough to matter
  scientifically rather than operationally

Phase `01.2.3` did not open a new scientific branch.
It closed the same-binary bundled `G2` route question and leaves one bounded
recovery-class question: whether the historical bundled `G2` residue belongs to
a neighboring launcher generation or requires explicit redevelopment.

## Current Battery Surface

The following remain battery, lane, or execution-variant questions:

- Mac CPU micro, medium, or long battery
- fresh RM10 CPU micro, medium, or long battery
- RM10 GPU parity battery
- RM10 NPU feasibility check
- heterogeneous embodiment experiments
- launcher, wrapper, cwd, env, and asset variations

## Phase-Specific Reminder

- A same-binary retry is not a new branch; it is closed work.
- A neighboring-launcher recovery question is not a battery variation; it is a
  separate bounded provenance or redevelopment decision.
- GPU, NPU, and heterogeneous work are implementation tracks, not branch
  promotions.

## Anti-Inflation Rule

Do not call GPU, NPU, or heterogeneous execution "the next branch" merely to
sound larger.
If the current blocker is launcher-generation versus redevelopment, widening the
hardware story before that blocker is settled is theater, not branching.
