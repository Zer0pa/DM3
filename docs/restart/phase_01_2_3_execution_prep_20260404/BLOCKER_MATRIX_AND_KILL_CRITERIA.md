## Lane-Specific Kill Criteria

### RM10 CPU witness lanes

Stop if:

- default validation is still unresolved and a longer battery would only add
  ambiguous receipts
- a `prebuilt_stub` or `mixed_prebuilt_backed` lane is being narrated as
  source-built parity
- thermal stress and drift cannot be localized

### Archaeology / launcher-provenance work

Stop if:

- the next step would require rewriting missing launcher or routing logic
- the work drifts from provenance into rebuild while still being called
  archaeology
- strings residue is being substituted for live launcher evidence

### RM10 GPU

Stop if:

- the CPU reference observable is not named explicitly
- parity rules are not explicit
- the path changes the observable family instead of accelerating it
- opaque vendor behavior blocks receiptable comparison

### RM10 NPU

Stop if:

- only hardware-adjacent presence is visible
- no callable user-space path exists
- inputs or outputs cannot be receipted
- the role drifts from bounded assist into opaque substitution

### Heterogeneous

Stop if:

- the split erases the comparable observable
- the CPU/GPU/NPU handoff cannot be named and logged
- drift cannot be localized to one side of the split
- the mixed lane changes the scientific story
