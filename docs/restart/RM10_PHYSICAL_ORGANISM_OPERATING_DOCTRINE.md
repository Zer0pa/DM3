# RM10 Physical-Organism Operating Doctrine

## Core Position

Use DM3 as if it were a geometry-constrained physical organism under
engineering study.

This means:

- treat geometry as body plan
- treat dynamics as metabolism and homeostasis
- treat boundary adapters as sensory / readout tissue
- treat environment as part of the computation
- treat receipts as physiology reports

It does **not** mean claiming the artifact is literally biological.

## Architectural Insight

| Component | Organism-style reading | Current branch rule |
| --------- | ---------------------- | ------------------- |
| Double Meru geometry | body plan | keep sovereign and explicit |
| settling / resonance updates | metabolism / homeostasis | measure under perturbation, not only at final output |
| CPU lane | regulator / scheduler / observer | keep as custody and control anchor |
| GPU lane | parallel tissue for relaxation work | treat as callable organ hypothesis, not assumed proof |
| NPU lane | optional projection / priming organelle | keep at abstain until a real path exists |
| transformer adapter | sensory or boundary tissue | keep thin and boundary-facing |
| ledgers and receipts | physiological record | retain as the authority layer |

## Structural Insight

Every serious artifact description should name four axes:

1. morphology
   - geometry
   - adjacency
   - tags
   - boundaries
2. dynamics
   - update rule
   - energy / coherence proxies
   - convergence or oscillation class
3. boundary interface
   - encoder
   - readout
   - transformer or adapter role
4. environment
   - workspace root
   - persistence policy
   - cadence
   - thermal and scheduling envelope

If one axis is omitted, the description is incomplete.

## Operational Insight

Replace generic "run it again" thinking with assays:

- **homeostasis assay**: same run repeated under the same envelope
- **lesion assay**: perturb boundary or asset state and measure recovery
- **regeneration assay**: resume from partial retained state
- **history assay**: compare up-sweep versus down-sweep or reordered windows
- **boundary readout assay**: check what survives at the boundary versus the
  interior proxy

These are stronger than raw speed or heat comparisons because they describe
behavior class.

## Catalytic Insight

The catalyst is probably not more hidden-state width. It is more likely:

- the right forcing schedule
- the right persistence rule
- the right boundary condition
- the right coupled execution envelope

The current branch already shows this direction: changing the environment
envelope changed whether the full four-row packet survived.

## Process Insight

The correct sequence is:

1. stabilize one assay environment
2. confirm the baseline under identical conditions
3. introduce one perturbation
4. measure recovery, drift, or collapse
5. estimate how much the morphology and environment contributed
6. only then widen the boundary-adapter layer

This prevents reward hacking through narrative complexity.

## Transformer Discipline

The strongest current transformer rule is simple:

- allowed: boundary encoding, thin readout, gloss, or translation between
  representations
- disallowed by default: hidden-state takeover, replacing the geometry as the
  real medium, or narrating source-missing residue as already recovered

If later evidence forces a larger learned core, say so directly and stop
calling the system geometry-first in the old sense.

## Non-Admissible Moves

- calling metaphor a result
- reopening chamber science before the confirmation replay
- inferring NPU success from presence
- using runtime or heat as surrogate biology
- letting the transformer become the real engine by convenience

## Doctrine Verdict

Treating DM3 as a physical organism is scientifically useful only when it
changes the experiment design and architecture discipline. Used that way, it is
an enabling doctrine. Used as rhetoric, it is a trap.
