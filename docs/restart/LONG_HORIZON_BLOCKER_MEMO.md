# DM3 Long-Horizon Blocker Memo

## Exact Phase Verdict

- `phase_outcome=PASS`
- `route_outcome=FAIL`
- The preserved bundled G2 family is retired on the current callable same-binary surface.
- The blocker is no longer "which same-binary route should we try?"; it is whether the historical bundled G2 residue belongs to a neighboring launcher generation or requires explicit redevelopment.

## Why This Is The Real Blocker Now

1. Phase `01.2.3` already exhausted the justified same-binary invocation
   surface and closed it exactly as retirement, not abstention.
2. The historical bundled residue still proves that a richer `G2 Boundary
   Readout / R2Contrastive` family once existed nearby, but the current callable
   same-binary surface does not expose it.
3. That leaves only two honest possibilities: the residue belongs to a
   neighboring launcher generation, or recovering it now would require explicit
   redevelopment.
4. Fresh RM10 Genesis lanes still remain governed but non-sovereign, with
   unresolved default-validator reconciliation and no source-built parity
   promotion, so witness-floor cleanup cannot substitute for this blocker.

## Why The Blocker Matters

- Gate D style hybrid comparison cannot advance from a retired same-binary
  route.
- GPU work cannot treat the retired bundled `G2` family as its reference
  observable.
- NPU and heterogeneous work cannot be inflated into recovery just because the
  same-binary question closed negatively.

## Re-entry Conditions

Only the following changes this blocker class:

- bounded evidence that the historical bundled `G2` residue belongs to a
  neighboring launcher generation with a live, receiptable surface
- an explicit decision that further progress requires redevelopment, with that
  redevelopment separated from archaeology rather than smuggled into it

## Later-Lane Consequences

- RM10 GPU is not ready now; it may start only if a different already-live CPU
  reference observable is named explicitly under the governed witness floor.
- RM10 NPU remains feasibility-only; a clean negative reachability result is a
  valid completion.
- Heterogeneous embodiment remains premature until a single-lane CPU reference
  observable is live, GPU parity has already passed on that same observable,
  and NPU feasibility is already bounded.
