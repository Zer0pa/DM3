# RM10 Geometric Organism System Synthesis

## Executive Statement

The current best evidence-backed reading of DM3 is:

`a geometry-constrained dynamical medium with optional learned boundary
adapters, not a plain transformer stack and not just a static geometry file.`

The RM10 branch is currently trying to learn how that medium behaves on a
real SoC execution surface without inventing missing source or widening claims
past the retained evidence.

## What Survives Strongly

### 1. Geometry

The strongest source-backed asset is the Double Meru geometry itself:

- high-symmetry structure
- adjacency and region / boundary organization
- exactness and proof-harness language

This is still the irreplaceable part of the artifact family.

### 2. Dynamics

The recovered code and operating doctrine strongly support:

- settling / DEQ-style update logic
- resonance / contraction language
- proof and receipt discipline
- energy- or stability-style acceptance thinking

### 3. Hybrid Evidence

The later compiled residue shows evidence of:

- GPU-bearing execution surfaces
- micro-transformer / HRM-adjacent language
- tagging / phoneme assets

But that source does not survive. This means the hybrid layer is real as
evidence, but not yet authoritative as architecture.

## How The Artifact Is Best Split

| Layer | Current role | Branch status |
| ----- | ------------ | ------------- |
| Geometry | body plan / state space | strongest source-backed layer |
| Dynamics | field settling / homeostasis | source-backed in older stacks, partial in live residue |
| Learned modules | boundary encoding, readout, bridge hypotheses | plausible, but source-missing as a modern stack |
| Acceptance | receipts, ledgers, abstain rules, falsifiers | strongest current governance layer |

## Where Transformers Fit Right Now

The current strongest line remains:

- defensible role: boundary adapter, encoder, readout, gloss, bridge
- non-defensible role: hidden-state takeover that makes geometry ornamental

If the transformer becomes the real hidden-state substrate, the project stops
being DM3-first and becomes a transformer with geometric decoration. The
current recovery does not justify that move.

## What The Branch Is Actually Doing

The branch is not proving a theory by metaphor. It is doing four narrower
things:

1. using the RM10 as a real instrument rather than a concept
2. separating the governed witness lane from the hybrid residue lane
3. testing whether environment and coupled execution change behavior class
4. keeping every widened claim behind exact retained packets

## What The Current Truth Floor Is

- geometry-first substrate remains the strongest real layer
- transformer / HRM-like continuation remains a bounded hypothesis
- same-family chamber widening once collapsed, then a stronger envelope
  repaired one four-row packet
- repair mechanism is unresolved
- NPU remains blocked
- explicit heterogeneous role partition remains blocked
- the immediate confirmation replay remains the live empirical gate

## Current Best Interpretation

The artifact looks less like a finished software model and more like a
geometry-constrained field process whose behavior depends on:

- body plan
- update dynamics
- boundary conditions
- environment and persistence
- the physical coupling envelope of the execution surface

That is why the "organism" frame is useful. It directs attention to
homeostasis, repair, persistence, and boundary exchange instead of only to
model weights or one-shot outputs.

## What This Synthesis Forces

- keep geometry sovereign
- keep the transformer thin until evidence justifies more
- treat environment as part of the system, not a nuisance
- treat repeated perturb-and-relax windows as first-class experiments
- preserve the confirmation replay as the next live gate
