# Roadmap: DM3 2026 Restart

## Overview

This roadmap takes DM3 from manifesto-driven narrative back to source-backed, falsifiable research. The sequence is: map claims, recover the deterministic substrate, define the authority gate, build the RM10 Pro device lane, run the cross-platform falsification battery, and only then decide what deserves architectural continuation.

## Contract Overview

| Contract Item | Advanced By Phase(s) | Status |
| ------------- | -------------------- | ------ |
| Manifesto test registry and gap map | Phase 1 | Planned |
| Source-backed deterministic baseline | Phase 2 | Planned |
| Explicit execution and authority contract | Phase 3 | Planned |
| RM10 Pro replay lane | Phase 4 | Planned |
| Cross-platform falsification battery | Phase 5 | Planned |
| Surviving-claims decision and rebuild boundary | Phase 6 | Planned |

## Phases

- [ ] **Phase 1: Claim Map and Contract Reset** - map `T01–T236` to real evidence or explicit gaps, and freeze what the restart is actually allowed to claim
- [ ] **Phase 2: Source-Backed Baseline Recovery** - rebuild and replay the recoverable October / Genesis deterministic substrate on Mac
- [ ] **Phase 3: Execution and Acceptance Specification** - define the manifold contract, authority metric, and minimal restart battery
- [ ] **Phase 4: RM10 Pro Execution Lane** - establish the Termux / ADB replay lane with receipts and Comet logging
- [ ] **Phase 5: Cross-Platform Falsification Battery** - rerun the selected source-backed battery on Mac and RM10 Pro and record drift or survival
- [ ] **Phase 6: Surviving Claims and Rebuild Decision** - decide which claims survive and whether any hybrid rebuild is justified

## Phase Details

### Phase 1: Claim Map and Contract Reset

**Goal:** Turn the manifesto from narrative into a governed claim inventory
**Depends on:** Nothing
**Requirements:** [MAPP-01, MAPP-02, MAPP-03]
**Contract Coverage:**
- Advances: manifesto test registry, false-proxy rejection
- Deliverables: mapped test matrix, gap register, contract reset note
- Anchor coverage: manifesto, recovery matrix, recovered testing protocol
- Forbidden proxies: prose-only evidence, visual demos, inherited certainty
**Success Criteria** (what must be TRUE):

1. Every manifesto test label used downstream is classified as mapped, collapsed, unmapped, or retired
2. Every mapped claim has a concrete source locator or protocol reference
3. The restart contract explicitly blocks citing unmapped claims as evidence
   **Plans:** 2 plans

Plans:

- [ ] 01-01: Build the manifesto test-reference matrix with source locators and gaps
- [ ] 01-02: Freeze the skeptical restart contract and carry-forward set
