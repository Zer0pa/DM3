**Track record:** Historical bundled outputs preserve nontrivial epoch tables, boundary-readout labels, and GPU dispatch traces. Fresh bundled calls still collapse to smoke, so the phase is not proving a winner in advance; it is testing whether explicit asset pairing resolves the collapse.

**Key steps:**

1. Freeze attempt identity before running:
   - binary: `/data/local/tmp/dm3/dm3_runner`
   - working directory: `/data/local/tmp/dm3`
   - observable family: `G2 Boundary Readout Experiment`

3. Prove the launch surface still exists:
   - `adb shell 'cd /data/local/tmp/dm3 && ./dm3_runner --help'`
4. Run the single explicit-asset attempt:

```bash
adb shell '
cd /data/local/tmp/dm3 &&
/system/bin/timeout 60 ./dm3_runner \
  --mode inference \
  --task exp_g2_readout \
  --adj SriYantraAdj_v1.bin \
  --tags RegionTags_v2.json \
  --output phase0122_g2_boundary_receipts.jsonl
```

6. Compare the fresh run against two reference surfaces only:

   - copy the untouched bundle into a fresh staging directory under `/data/local/tmp`
