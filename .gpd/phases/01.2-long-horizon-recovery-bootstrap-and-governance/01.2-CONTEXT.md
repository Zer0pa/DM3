# Phase 01.2 Context: Long-Horizon Recovery Bootstrap and Governance

## Mission

Execute the control phase required by the long-horizon DM3 mandate before wider
baseline replay or hybrid escalation continues.

This phase does not claim the whole mandate is finished.
It establishes the governed working surface from which the later phases can be
executed honestly.

## Operating Law

1. Source-backed evidence outranks narrative.
2. The top authority metric remains sovereign over local engineering wins.
3. Governed Genesis / SoC and later DM3 hybrid residue stay split.
4. RM10 CPU is the first safe phone lane.
5. RM10 GPU is an engineering lane only after parity rules are explicit.
6. NPU remains feasibility-only until a callable, receiptable user-space path
   is proven.
7. Bounded repair is allowed; hidden rebuild is forbidden.

### Gate A: custody and architecture map

Answer:

- what survives
- how many hybrid generations there were
- what the strongest authority surfaces are
- what the next runnable surface is

### Gate B: canonical and contamination split

Answer:

- what drift is contamination
- what drift is real divergence
- what canonical target is sovereign for each lane
- which receipt fields are clean enough for comparison

Answer:

- whether one fresh non-stub task was recovered

- the narrowest blocker

Answer:

- whether GPU recovery is real and receiptable
- whether it remains smoke only
- whether CPU/GPU comparison can be done honestly

Answer:

- which scientific story survives best right now:
  - geometry-first field computer
  - structure-first memory substrate
  - boundary hybrid
  - nodewise hybrid
  - center-model hybrid
  - narrower ordinary hybrid
