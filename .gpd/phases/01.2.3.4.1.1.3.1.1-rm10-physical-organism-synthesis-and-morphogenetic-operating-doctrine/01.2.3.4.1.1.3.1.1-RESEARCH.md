**Confidence:** MEDIUM

## Summary

The strongest evidence-backed reading of the current branch is that DM3 is not
best treated as "a better transformer" and not best treated as a mystical
geometric claim. It is better treated as a geometry-constrained dynamical
medium whose behavior depends on body plan, environment, perturbation schedule,
and boundary coupling. In that frame:

- geometry is the body plan
- settling / resonance / DEQ-style evolution is the metabolism or homeostatic
  dynamics
- CPU/GPU/NPU roles are organ or tissue hypotheses, not inherited facts
- transformer-like machinery is strongest as a boundary adapter hypothesis
- current RM10 work is really probing whether the physical execution envelope
  changes the behavior class of that medium

The external literature does not prove DM3. It does, however, sharpen the
right vocabulary and experiment design. Morphogenesis and bioelectric work
support treating distributed matter as a target-seeking, error-correcting
system with memory and control loops. Physical reservoir computing supports
using nonlinear history-bearing substrates as computational media, especially
on edge devices. Morphological computation supports measuring how much the body
and environment contribute relative to explicit controller action.

**Primary recommendation:** formalize DM3 as a physical-organism-style
computation doctrine for branch work. Keep the immediate confirmation replay as
the live gate, then extend measurement toward homeostasis, regeneration,
history dependence, and morphological contribution rather than only one-shot
output interpretation.

### Why The Organism Frame Helps

The branch already discovered three things that a pure software-model frame

- environment matters
- repeated windows matter

## External Literature Bridge

### Geometric Computation

**Kratsios and Papon, "Universal Approximation Theorems for Differentiable
Geometric Deep Learning" (2021)** argues that geometry-compatible models on
manifolds can approximate broad target classes, but also that purely local
geometric models have real representational limits across manifolds. Source:
[arXiv:2101.05390](https://arxiv.org/abs/2101.05390).

**Useful carry-back:** if DM3 claims that geometry matters, the geometry cannot
become decorative. The geometry needs to remain an explicit hard structure with
global transport and boundary behavior visible in the design. This supports the
current branch line that transformer-like modules should stay thin and
boundary-facing unless evidence proves otherwise.

### Reaction-Diffusion And Pattern Formation

**Mordvintsev, Randazzo, and Niklasson, "Differentiable Programming of
Reaction-Diffusion Patterns" (2021)** treats reaction-diffusion systems as a
computational framework that can be optimized and can display robust, nontrivial,
"life-like" behavior. Source:
[arXiv:2107.06862](https://arxiv.org/abs/2107.06862).

**Grodstein and Levin, "Closing the Loop on Morphogenesis" (2022)** models
morphogenesis as closed-loop negative feedback with computation waves that scan
and regulate a target pattern. Source:
[arXiv:2211.01313](https://arxiv.org/abs/2211.01313).

**Useful carry-back:** DM3 should be instrumented as a target-seeking,
distributed control system, not just as a generative black box. The right
questions are therefore about:

- homeostasis after perturbation
- regeneration after boundary damage
- whether history and direction of perturbation matter
- whether readout requires a scanning or boundary wave, not only a final hash

### Biocomputation And Basal Cognition

**Levin, "Technological Approach to Mind Everywhere (TAME)" (2021)** frames
morphogenesis as a form of embodied, multiscale problem-solving and emphasizes
developmental bioelectricity as a medium for scaling local feedback loops into
anatomical homeostasis. Source:
[arXiv:2201.10346](https://arxiv.org/abs/2201.10346).

**Useful carry-back:** the branch should treat DM3 as a system of local
competencies that may scale into a larger behavior only through boundary
conditions, persistent state, and distributed feedback. This makes the
"organism environment" language operational rather than poetic.

### Morphological Computation

**Zahedi and Ay, "Quantifying Morphological Computation" (2013)** formalizes
how much the morphology and environment contribute to behavior relative to the
agent's action. Source:
[arXiv:1301.6975](https://arxiv.org/abs/1301.6975).

**Mertan and Cheney, "No-brainer: Morphological Computation driven Adaptive
Behavior in Soft Robots" (2024)** shows adaptive behavior can emerge from the
physical body without a separate heavy controller. Source:
[arXiv:2407.16613](https://arxiv.org/abs/2407.16613).

**Useful carry-back:** current DM3 work should stop assuming that usefulness
must appear only when the explicit learned controller grows. The question
should become: how much useful computation is already being carried by the
geometry, environment, and coupled dynamics?

### Physical Reservoir Computing

**Nakajima, "Physical reservoir computing -- An introductory perspective"
(2020)** explains how complex physical dynamics with fading memory can serve as
computational devices, especially in edge settings. Source:
[arXiv:2005.00992](https://arxiv.org/abs/2005.00992).

**Wang and Cichos, "Harnessing Synthetic Active Particles for Physical
Reservoir Computing" (2023)** shows that self-organized noisy active systems
can compute when read out through historical states. Source:
[arXiv:2307.15010](https://arxiv.org/abs/2307.15010).

**Useful carry-back:** the live RM10 branch should think in terms of:

- fading memory
- nonlinear response
- history-bearing reservoir state
- boundary readout over short history windows
- self-organization plus noise rejection

That is a much better fit than insisting that every useful signal must look
like deterministic one-shot parity immediately.

## Distilled Insight Families

### Architectural Insight

The best current architecture is:

- geometry as body plan
- dynamics as homeostatic field evolution
- boundary adapters as sensory / readout tissue
- acceptance ledgers as physiology reports

The branch should resist converting this into "transformer core with geometry
decoration" unless later evidence forces that conclusion.

### Structural Insight

The artifact should be modeled along four independent axes:

1. morphology: mesh, adjacency, boundary and region tags
2. dynamics: settling, resonance, energy / coherence proxies, state update
3. boundary interfaces: encoders, readouts, transforms, transformer adapters
4. environment: workspace, persistence, cadence, thermal envelope, handoff

This means experiment records should always name all four axes.

### Operational Insight

The right assay language is:

- homeostasis
- regeneration
- hysteresis
- lesion and repair
- transplantation / resume
- boundary readout fidelity

That is stronger than "CPU run versus GPU run" because it describes the class
of behavior, not just the lane.

### Catalytic Insight

The catalyst is probably not a bigger central model. It is the right forcing
schedule, persistence policy, and coupling envelope. This is where the RM10
work already points: changing row order, envelope, and window structure changes
what survives.

### Process Insight

Branch work should proceed in this order:

1. stabilize the assay environment
2. measure homeostasis under repeated identical runs
3. introduce one perturbation at a time
4. classify regeneration or collapse
5. quantify how much the body / environment contributed
6. only then try richer boundary adapters

## What This Does Not Justify

- It does not justify reopening heterogeneous success claims.
- It does not justify calling the artifact biologically alive.
- It does not justify a new transformer-centered rebuild.
- It does not justify skipping the confirmation replay.

## Research Verdict

The organism frame is useful and technically disciplined if it is cashed out as
a homeostatic, morphogenetic, and reservoir-computing doctrine. It becomes
harmful only if used as a substitute for receipts or as a shortcut to chamber
claims. The immediate branch gain is better experimental design and better
architecture discipline, not a widened scientific verdict.
