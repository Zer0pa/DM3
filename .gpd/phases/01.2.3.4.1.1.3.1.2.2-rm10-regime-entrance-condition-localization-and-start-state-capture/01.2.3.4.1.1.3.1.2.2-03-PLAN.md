# Plan 03: Compare Anchors And Classify The Strongest Entrance-Condition Candidate

<objective>
Compare all anchor packets across both axes (session speed and value regime),
identify whether deep-clean, shallow-clean, or no-clean produces a regime
difference, and classify the strongest entrance-condition candidate.
</objective>

## Tasks

### Task 1: Build the comparison table

For each anchor, extract and tabulate:

| Anchor | Clean type | Idle | Residual count | delta_E | coherence | duration_ms | Regime | Speed |
|--------|-----------|------|---------------|---------|-----------|-------------|--------|-------|
| A | deep | 120s | 0 | ? | ? | ? | ? | ? |
| B | none | 0s | N | ? | ? | ? | ? | ? |
| C | shallow | 0s | N-1 | ? | ? | ? | ? | ? |
| D | deep | 10s | 0 | ? | ? | ? | ? | ? |

### Task 2: Test hypothesis H1 (residual files)

Compare A vs B vs C:
- If A differs from B/C on regime → residual files matter
- If A = B = C → residual files are NOT the selector

### Task 3: Test hypothesis H2 (filesystem cache / idle duration)

Compare A vs D (both deep-clean, different idle):
- If A differs from D → idle/cooldown matters (cache eviction, governor reset)
- If A = D → idle duration is NOT the selector on its own

### Task 4: Test session stability

Compare B vs C (both run after A, different clean levels):
- If B = C → within-session regime is stable regardless of output cleanup
- If B != C → the presence of the specific output file matters

### Task 5: Compare against prior packets

Map each anchor to the prior packet regimes:
- Does any anchor match the repaired packet's fast-session HIGH profile?
- Do all anchors match the localization's slow-session LOW profile?
- Is there a novel combination not seen before?

### Task 6: Classify the strongest candidate

Based on the comparisons, identify:
1. The most discriminating pre-run variable
2. Whether the session-speed axis and value-regime axis are independently controllable
3. Whether any tested condition reliably enters the HIGH regime

If no tested condition enters HIGH:
- Record what was ruled out
- Identify what remains untested (device reboot, binary pre-warming, explicit
  filesystem cache warming, etc.)
