# Plan 01: Freeze Entrance-Condition Contract And Start-State Capture Bundle

<objective>
Define the exact pre-run state capture protocol, the deep-clean versus shallow-clean
distinction, the locked envelope for single-row cpu_a anchors, and the comparison
tuple that will classify entrance-condition candidates.
</objective>

## Tasks

### Task 1: Write the entrance-condition observable contract

Define exactly what distinguishes a HIGH-regime from a LOW-regime cpu_a run:

| Metric | HIGH regime | LOW regime |
|--------|-------------|------------|
| delta_E | ~88-89 | ~75-76 |
| coherence | ~0.77 | ~0.877-0.893 |
| duration_ms | ~155,000 (fast session only) | ~200,000 |

Note: duration appears session-level while values appear row-level. Both must be
captured.

### Task 2: Define the deep-clean protocol

Before each "cold" anchor:
1. `rm -f /data/local/tmp/*.jsonl` (remove ALL output residuals)
2. `rm -f /data/local/tmp/codex_phase_probe_*.jsonl`
3. Verify no dm3_runner process: `ps -A | grep dm3_runner`
4. Wait 120 seconds
5. Capture full pre-state bundle (Task 3)

Before each "shallow-clean" anchor:
1. `rm -f /data/local/tmp/<specific_output_file>.jsonl` only
2. No wait
3. Capture full pre-state bundle (Task 3)

Before each "no-clean" anchor:
1. No removal at all, use a different output filename
2. No wait
3. Capture full pre-state bundle (Task 3)

### Task 3: Define the pre-state capture bundle

Retain BEFORE each anchor run:
- `adb shell dumpsys battery` (battery snapshot)
- `adb shell dumpsys thermalservice | head -60` (thermal snapshot)
- `adb shell cat /proc/loadavg` (system load)
- `adb shell ps -A | grep dm3_runner` (process snapshot)
- `adb shell sha256sum /data/local/tmp/dm3_runner` (binary hash)
- `adb shell 'ls -la /data/local/tmp/*.jsonl /data/local/tmp/*.bin 2>/dev/null'` (sidecar listing)
- `adb shell 'ls -la /data/local/tmp/data/ 2>/dev/null'` (dataset listing)
- `adb shell cat /proc/stat | head -10` (CPU time counters)

### Task 4: Define the locked single-row envelope

Identical across all anchors:
- binary: `/data/local/tmp/dm3_runner`
- cwd: `/data/local/tmp`
- command: `cd /data/local/tmp && /system/bin/timeout 420 /data/local/tmp/dm3_runner --mode train --task harmonic --steps 1 --cpu --output /data/local/tmp/<anchor_output_name>.jsonl`
- row timeout: 420 seconds
- no `--gated`, no `--calibration`, no explicit `--tags` or `--adj` (use binary defaults)

### Task 5: Write the comparison tuple

For each anchor, retain and compare:
- regime classification: HIGH / LOW / INDETERMINATE
- session speed: FAST (~155s) / SLOW (~200s)
- pre-state: clean type (deep/shallow/none), idle duration, residual file count, thermal, battery, loadavg
- post-state: output content, duration_ms, delta_E, coherence
