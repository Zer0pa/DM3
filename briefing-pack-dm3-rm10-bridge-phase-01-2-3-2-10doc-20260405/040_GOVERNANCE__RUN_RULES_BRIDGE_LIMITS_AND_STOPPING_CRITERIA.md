# Governance

## Current Non-Negotiables

- do not let `F2` speak for `F1`
- do not count accelerator presence as a callable lane
- do not count opaque in-binary GPU use as explicit heterogeneous role partition
- do not claim an NPU lane without a named user-space command and receipted I/O
- do not recast `ABSTAIN`, `bridge_closed`, or `unstable_feasibility` as soft wins

## Updated Bridge Rule

Bridge promotion now requires all of the following:

1. a same-family accelerator-bearing surface on `F1`
2. the preserved governed observable under that surface
3. explicit stage ownership if heterogeneous execution is claimed
4. pre-handoff and post-handoff artifacts

Anything less stays below bridge grade.

## Current Stop Conditions

- more residue-only GPU rows without a sharpened question
- renewed NPU enthusiasm without a callable command
- handoff claims without localized artifacts
- any attempt to treat the pack itself as evidence of scientific strengthening
