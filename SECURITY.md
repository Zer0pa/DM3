# Security Policy

## Reporting

Report security vulnerabilities, credential leaks, or evidence-tampering
concerns to `architects@zer0pa.ai`.

Include:

- affected path or artifact
- reproduction steps
- scope of exposure
- any proposed containment action

## Expectations

- Do not open public issues for undisclosed vulnerabilities.
- Do not alter or delete evidence artifacts while investigating a report.
- Treat repo integrity problems and proof-surface corruption as security issues.

The root [LICENSE](LICENSE) file is the legal source of truth for this repo's
licensing posture.
